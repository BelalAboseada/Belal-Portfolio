# Performance & Architecture Optimization Plan — Belal Portfolio

## Context

This is a Next.js 16 portfolio site with 12 client components using GSAP for heavy animation, Tailwind CSS v4 for styling, and Framer Motion (bundled but unused in visible code). All sections render eagerly on the single-page route, with inline GSAP ScrollTrigger animations, custom cursors, marquee loops, and SVG noise filters.

---

## Phase 1 — Critical Performance Fixes (High Impact, Low Effort)

### 1.1 Fix Redundant GSAP ScrollTrigger Registrations

**Location**: `About.tsx:18`, `Projects.tsx:49`, `Skills.tsx:14`, `ContentCreator.tsx:15`, `Contact.tsx:15`

Each component calls `gsap.registerPlugin(ScrollTrigger)` inside its own `useEffect`. This is already handled once globally in `gsap-init.ts:6`. Redundant calls add overhead on every mount.

**Fix**: Remove all component-level `gsap.registerPlugin(ScrollTrigger)` calls. Plugin registration is a one-time global operation.

**Affected files**: `About.tsx`, `Projects.tsx`, `Skills.tsx`, `ContentCreator.tsx`, `Contact.tsx`

---

### 1.2 Replace Inline Animated `<style>` Tag in HeroMask

**Location**: `HeroMask.tsx:205-219`

A `<style>` element with CSS keyframes is injected per mount. CSSOM insertions on every mount cause style recalculations.

**Fix**: Move `fadeInY`, `fadeInY10`, `fadeIn`, `fadeInScale`, and `blink` keyframes into `globals.css`, reference them by class name.

**Affected files**: `HeroMask.tsx`, `globals.css`

---

### 1.3 Replace Per-MouseMove `gsap.to` with `gsap.quickSetter` in Cursor

**Location**: `Cursor.tsx:32`

`gsap.to(cursor, { x, y, duration: 0 })` creates a new tween object on every pixel of mouse movement. Equivalent to hundreds-to-thousands of tween allocations per second.

**Fix**: Replace with `gsap.quickSetter` (already used for follower — apply same pattern to main cursor).

```ts
const cursorXSet = gsap.quickSetter(cursor, "x", "px");
const cursorYSet = gsap.quickSetter(cursor, "y", "px");
// In onMouseMove: cursorXSet(e.clientX); cursorYSet(e.clientY);
```

**Affected files**: `Cursor.tsx`

---

### 1.4 Prerender or Remove SVG Noise Filter

**Location**: `HeroMask.tsx:222-229`

`<feTurbulence type="fractalNoise" numOctaves="3">` is a GPU-intensive SVG filter that runs every frame. It covers the entire viewport with opacity 0.03, providing negligible visual value at high cost.

**Fix**: Either:
- Remove it entirely (preferred — the opacity is 3%, near-invisible)
- Replace with a static pre-generated noise PNG/CSS pattern

**Affected files**: `HeroMask.tsx`

---

### 1.5 Eliminate HeroMask Scroll Event Listener Tween Allocation

**Location**: `HeroMask.tsx:174-188`

On every scroll event (dozens per second), this creates a new `gsap.to()` tween for the scroll indicator. Even with `overwrite: "auto"`, this generates excessive GC pressure.

**Fix**: Use a simple boolean toggle with a CSS transition instead of GSAP for this one-element opacity toggle, or at minimum debounce/throttle the handler to ~100ms.

**Affected files**: `HeroMask.tsx`

---

### 1.6 Remove Inline Hover GSAP Tweens in HeroMask CTA

**Location**: `HeroMask.tsx:314-323`

`onMouseEnter`/`onMouseLeave` create 3 new GSAP tweens each (6 total per hover cycle). For simple transform/opacity transitions, CSS transitions are free.

**Fix**: Replace with CSS transition classes (`transition-all duration-300`) or Tailwind `group-hover:` utilities.

**Affected files**: `HeroMask.tsx`

## Phase 2 — Bundle & Loading Optimization

### 2.1 Lazy-Load Below-Fold Sections

**Location**: `page.tsx:14-29`

All 8 sections (including 4 commented-out) render eagerly. Below-fold sections (Projects, Skills, Contact, Footer) consume bundle budget and hydration time for content the user won't see immediately.

**Fix**: Use `next/dynamic` to lazy-load client components:

```tsx
const About = dynamic(() => import("@/components/About"));
const Projects = dynamic(() => import("@/components/Projects"));
const Skills = dynamic(() => import("@/components/Skills"));
const Contact = dynamic(() => import("@/components/Contact"));
const Footer = dynamic(() => import("@/components/Footer"));
```

Add lightweight skeleton/loading states for each section.

**Affected files**: `page.tsx`

---

### 2.2 Configure next.config.ts for Production

**Location**: `next.config.ts`

Currently empty. Add essential production configs:

- **Image optimization**: Configure `images.formats` (avif, webp), `images.deviceSizes`
- **Bundle analysis**: Enable `@next/bundle-analyzer` config
- **Compiler**: Remove console.log in production (`compiler.removeConsole`)
- **Experimental**: Enable `optimizePackageImports` for `gsap`, `framer-motion`, `lucide-react`

**Affected files**: `next.config.ts`

---

### 2.3 Migrate Google Fonts to next/font

**Location**: `layout.tsx:18-23`

Currently using `<link>` tags to Google Fonts CDN. This means:
- No self-hosting (network dependency on Google)
- No font subsetting
- Cumulative Layout Shift (CLS) during font swap

**Fix**: Replace with `next/font/google`:

```tsx
import { Cairo, Inter } from "next/font/google";
const cairo = Cairo({ subsets: ["arabic", "latin"], weight: ["400", "600", "700", "900"], variable: "--font-display", display: "swap" });
const inter = Inter({ subsets: ["latin"], weight: ["300", "400", "500", "600"], variable: "--font-body", display: "swap" });
```

This self-hosts fonts, subsets automatically, and eliminates CLS.

**Affected files**: `layout.tsx`, `globals.css`, `tailwind.config` (or Tailwind v4 theme)

---

### 2.4 Tree-Shake GSAP Imports

**Location**: All components importing GSAP

Currently importing `gsap` (full build, ~60KB gzipped) and `gsap/dist/ScrollTrigger` across 7 components. GSAP supports selective imports via `gsap/ScrollTrigger` paths.

**Fix**: Already using `gsap/dist/ScrollTrigger` which is correct for tree-shaking. Verify the main `gsap` import isn't pulling CSSPlugin unnecessarily, and remove `ScrollToPlugin` import from `gsap-init.ts` if anchor smooth-scroll is replaced by CSS `scroll-behavior: smooth`.

**Affected files**: `gsap-init.ts`

## Phase 3 — Runtime Performance & Architecture

### 3.1 Create Reusable Section & Card Components

**Location**: Multiple components

Identical patterns repeated across `About`, `Projects`, `Skills`, `ContentCreator`, `Clients`, `SocialFeed`:
- Card hover: `onMouseEnter` setting `boxShadow`, `onMouseLeave` resetting it
- SVG corner decoration on hover
- `bg-[#10101c] border border-[#1e1e35] rounded-xl` base style
- GSAP ScrollTrigger fade-in pattern identical across 4+ components

**Fix**: Extract:
- `<Section>` — shared section wrapper (backdrop, WebLines, container, GSAP entry animation)
- `<Card>` — shared card with hover shadow, border transition, corner SVG
- `useScrollReveal` hook — encapsulates the `gsap.from(..., { scrollTrigger })` pattern

**Affected files**: New `src/components/ui/` directory; refactor all section components

---

### 3.2 Extract Project Data to Config File

**Location**: `Projects.tsx:8-41`

Hardcoded project data in component blocks server-side rendering potential and makes the component harder to tree-shake.

**Fix**: Move to `src/data/projects.ts` (static export) or a headless CMS integration for dynamic content.

**Affected files**: `Projects.tsx`, new `src/data/projects.ts`

---

### 3.3 Add Error Boundaries for Animation Components

**Location**: All `"use client"` animation components

If GSAP fails (ad blockers, CSP, network issues), the entire page crashes with an unhandled error.

**Fix**: Wrap animated sections in a lightweight `<AnimationErrorBoundary>` that silently degrades to static content.

**Affected files**: `page.tsx`, new `src/components/AnimationErrorBoundary.tsx`

---

### 3.4 Debounce/Throttle Scroll-Driven Operations

**Location**: `Navbar.tsx:19`, `HeroMask.tsx:174`

The scroll event handlers fire at 60+ FPS. The Navbar handler does a simple `setScrolled(window.scrollY > 60)` which is fast (setState batching handles it), but the HeroMask handler creates GSAP tween allocations as noted in 1.5.

**Fix**: For Navbar, use `requestAnimationFrame` wrapper to avoid setState calls during non-paint frames. For HeroMask, use CSS `@media (prefers-reduced-motion)` or throttle.

**Affected files**: `Navbar.tsx`, `HeroMask.tsx`

---

### 3.5 Fix MarqueeText GSAP Timeline Accumulation

**Location**: `MarqueeText.tsx`

Used in `Projects.tsx` (banner), `Clients.tsx` (2 instances), `SocialFeed.tsx` (banner). Each creates a `gsap.timeline({ repeat: -1 })` that runs perpetually on the GSAP ticker. Multiple infinite timelines consume main thread even when sections are scrolled out of view.

**Fix**: 
- Use CSS `@keyframes` + `animation` for marquee instead of GSAP (modern CSS supports `translate` animation with `infinite` iteration)
- Or use `gsap.context()` with `ScrollTrigger` to pause/kill marquees when the section is not in viewport

**Affected files**: `MarqueeText.tsx`

---

### 3.6 Replace Inline `onMouseEnter`/`onMouseLeave` Shadow Manipulation with CSS

**Location**: `About.tsx:79-80`, `Projects.tsx:95-96`, `Skills.tsx:122-123`

Inline event handlers that directly mutate `e.currentTarget.style.boxShadow` are slower than CSS transitions and bypass Tailwind's optimization.

**Fix**: Use Tailwind's `group-hover:` variant with a parent `group` class and CSS variable-based shadow:

```css
.card { box-shadow: var(--shadow-card); transition: box-shadow 0.3s; }
.card:hover { box-shadow: var(--shadow-hover); }
```

**Affected files**: `About.tsx`, `Projects.tsx`, `Skills.tsx`

## Phase 5 — Monitoring & Maintenance

### 5.1 Synthetic Monitoring

Add Lighthouse CI to the build pipeline to track Core Web Vitals (LCP, FID/INP, CLS) across builds.

### 5.2 GSAP Version Audit

Lock GSAP to a specific version and regularly audit for plugin tree-shaking opportunities. The `ScrollToPlugin` import in `gsap-init.ts` may be unnecessary if smooth-scroll is handled by CSS `scroll-behavior: smooth`.

### 5.3 Remove Commented-Out Components or Mark Them for Future

Three components are imported but commented out in `page.tsx`:
- `ContentCreator` — 111 lines
- `Clients` — 51 lines  
- `SocialFeed` — 55 lines

These are still bundled if imported (the import statement itself adds them to the dependency graph). Remove the imports until actively needed, or use `next/dynamic` with lazy loading.

**Affected files**: `page.tsx`

## Estimated Impact

- **Bundle size reduction**: 40-60% (removing framer-motion, tree-shaking GSAP, lazy loading sections)
- **Runtime FPS improvement**: 25-40% (cursor optimization, noise filter removal, scroll handler fixes)
- **First Contentful Paint**: ~30% faster (font self-hosting, lazy loading)
- **Cumulative Layout Shift**: Near-zero (next/font with display:swap)
- **Time to Interactive**: ~25-35% faster (code splitting, reduced JS)
